# REQ — 요구사항 정의서  [③ 가이드]

> 기능·비기능 요건을 우선순위와 함께. 화면·정책·테스트와 ID로 추적.
> 우선순위 = MoSCoW: Must / Should / Could / Won't(이번 제외)

## 기능 요건
| 요건 ID | 구분 | 요구사항명 | 상세 | 우선순위 | 관련 화면·정책 | 검증(TC) |
| --- | --- | --- | --- | --- | --- | --- |
| *REQ-001* | *기능* | *키워드 검색* | *키워드 2자+ 입력 시 0.3초 내 결과 노출* | *Must* | *SCR-001 / POL-001* | *TC-001* |
|  |  |  |  |  |  |  |

| 요건 ID | 구분 | 요구사항명 | 상세 | 우선순위 | 관련 화면·정책 | 검증(TC) |
| --- | --- | --- | --- | --- | --- | --- |
| *REQ-101* | *접근성* | *VoiceOver/스크린리더 지원* | *모든 셀이 의미 있는 순서로 읽힘* | *Must* | *(전역)* | *TC-102* |
| *REQ-102* | *성능* | *응답 0.1초* | *주요 동작 0.1초 이내 UI 반영* | *Must* | *(전역)* | *TC-101* |
