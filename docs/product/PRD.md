# PRD — 서비스 기획서  [① 상시 브리프]

> 서비스의 단일 기준점. 무엇을·왜·누구를 위해 만드는지. 주 독자: 전원
> 표의 *기울임* 값은 예시 — 실제 내용으로 교체.
> 작성 가이드: `rules/methodology/templates.md` 의 ① PRD

## 기본 정의
| 항목 | 내용 |
| --- | --- |
| 서비스 한 줄 정의 | *「누구의 / 어떤 문제를 / 어떻게 해결하는」 서비스인지 한 문장* |
| 배경 · 문제 정의 | *왜 지금 필요한가. 사용자의 불편/기회를 구체적으로* |
| 타겟 사용자 | *핵심 페르소나와 사용 맥락(언제·어디서·어떤 상황)* |
| 핵심 가치 · 차별점 | *대안 대비 강점 1~3가지* |
| 마일스톤 | *기획 → 디자인 → 개발 → QA → 출시 일정* |
| 제약 · 전제 | *기술·예산·일정·정책 제약, 가정* |

## 범위 정의 (In / Out / Backlog)  ★ Out of scope가 진짜 핵심
| 구분 | 내용 |
| --- | --- |
| **In scope** (이번 포함) | *REQ-### ID와 함께. 이번 버전에 반드시 포함* |
| **Out of scope** (이번 안 함) | *의도적으로 제외한 것 + 이유. AI가 범위 넘는 걸 막는 칸* |
| **Backlog** (다음 버전) | *지금은 안 하지만 다음에. '안 함'과 '잊음'의 구분* |

| --- | --- | --- | --- |
| *주간 재방문율* | *WAU / 총가입자* | *30%* | *출시 3개월 내* |
